1.0 Getting Started

The ASSIST/UNA software is a Windows-based emulator of the ASSIST/I assembler for the IBM/360 machine. This software provides a graphical environment in which to write assembly language programs. The ASSIST/UNA software is meant to be used primarily by students enrolled in assembly language courses.

If you have concerns that require more detailed explanations, please consult the PDF user manual, which accompanied the distribution of ASSIST/UNA.

- 1.1 Starting a New Project

  1. Click the New button on the toolbar or select File → New. (Ctrl + N)
  2. This creates a new .una project file.

- 1.2 Saving Your Project

Your project must be saved before assembly can occur.
To save your project:

  1. Click the Save button on toolbar or select File → Save. (Ctrl + S)
  2. Choose a location for your saved project files.
  3. Choose a name for your project.

- 1.3 Opening Your Project

To open a previously saved project:

  1. Click the Open button on the toolbar or select File → Open. (Ctrl + O)
  2. In the Open File Dialog Window, choose which .una file you would like to open.
Opening a project will reset the ASSIST/UNA environment to its default state.

2.0 Assembling a Program

When assembling a program in ASSIST/UNA, it is vital that you understand the differences between each assembly option. This section covers how to most efficiently utilize each assembly option. It also covers how to view both the .PRT and the output window.

- 2.1 Assemble and Generate a .PRT File

The main purpose of this option is to allow you to quickly assemble your source code and check the .PRT file for any errors that may be present within your code.
Warning: Assembly does not generate any program output.
To begin Assembly:

  1. Click the Assemble button on toolbar or select Assemble → Assemble. (F2)
  2. The source code is assembled and a .PRT file is generated.
  3. The generated .PRT file will have the same name as your project file.
  4. Any errors found in the source code will be reported in the .PRT file.

- 2.2 Assemble and Debug

This option allows you to step through your program as it is simulated. This option generates program output.
Warning: This option temporarily generates a .PRT file before deleting it.
To begin Debugging:

  1. Click the arrow next to the Assembly button on the toolbar and select Assemble and Debug from the drop-down menu. Alternatively, you can select Assemble → Assemble and Debug in the top menu. (F3)
  2. After assembly has begun, use the Next button to step through each line. (F5)
  3. If you want to stop debugging, click the Stop button. (F8)
  4. If you would like to completely run through your program in debug mode, click Run To End. (F7)
  5. If you would like to run to a specific location, without executing the instruction, you may select the Run To Location option. (F6)
  6. As each statement is executed, both the Registers Display and the Memory Display will be updated.
  7. After assembly is completed, the Symbol Table Display will update and show the symbol table contents.
  8. All output generated by the program will be displayed in the Output Dialog Window. Any runtime exceptions will be reported in the temporary .PRT file and in the Output Dialog Window.

- 2.3 Assemble/Final Run

This option allows you to assemble and execute your source code. This option generates output and a .PRT file.
To begin a Final Run:

  1. Click the arrow next to the Assembly button on the toolbar and select Assemble/Final Run from the drop-down menu. Alternatively, you can select Assemble → Assemble/Final Run in the top menu. (F4)
  2. A text box will appear prompting you for an identifier. This identifier will appear at the top of your .PRT file.
  3. Once assembly completes, the symbol table contents will be updated.
  4. The program will completely assemble without interruption, regardless of whether or not the source program has errors.
  5. Once execution concludes, both the Registers Display and the Memory Display will be updated.
  6. Any output generated by the program will be displayed in the .PRT file and the Output Dialog Window. All errors will be reported in the both windows.

Important Note:

  • The maximum number of errors supported in the assembler is 300.
  • The maximum number of symbols in one program is 100.
  • The maximum number of literals: 50 + (50 * number of LTORGs).

- 2.4 View .PRT File

The .PRT file associated with your project will share the same name as your project file.
Reminder: Only the Assemble and Assemble/Final Run options will generate a permanent .PRT file. To view the .PRT file:

  1. Click the View .PRT button on the toolbar, or select Tools → View .PRT. (F9)
  2. If a .PRT has been generated, then a window will open containing it.
Important Note: An automatic TITLE is generated in the .PRT every 45 lines.

- 2.5 View Output Dialog Window

This displays all errors and output generated by the simulator.
Reminder: Only the Assemble and Debug and Assemble/Final Run options will produce program output. To view output:

  1. After using either Assemble and Debug or Assemble/Final Run, the output window will automatically be displayed.
  2. If you would like to manually view the output window, select Tools → Output . (F10)

3.0 Printing

This section covers printing both the source code and the .PRT file.

- 3.1 Print Source Code

By default, the source code will print in portrait mode.
To print the source code:

  1. Click the Print button on the toolbar, or select File → Print. (Ctrl + P)
  2. A Print Dialog box is displayed, allowing you to alter any print settings.

- 3.2 Print .PRT File

By default, the .PRT will print in Landscape mode.
To print the .PRT:

  1. First select the View .PRT option (see Section 2.4).
  2. Then, in the View .PRT Window, selet the Print button.
  3. If the .PRT file exists, then a Print Dialog box is displayed, allowing you to alter any print settings.

4.0 Editing Your Program

The most basic way to edit your program is to simply type some text into the source code editor. This section covers some of the different ways that ASSIST/UNA allows you to edit and navigate your source code.

- 4.1 Pasting Text into the Source Code Editor

To paste text into your program:

  1. After copying some text, then use either Ctrl + V or right-click → Paste to paste your copied text into the source code editor.
  2. Any text that is pasted into the source code editor will automatically be correctly formatted.

- 4.2 Search for Text in Source Code

This tool allows you to quickly search through your source code to find all occurences of some search term. To use Search:

  1. Select Edit → Search. (Ctrl + F)
  2. A text box will appear prompting you to enter a search term.
  3. After entering a search term, click Find to jump to its first occurrence within your source code.
  4. Continue clicking Find to cycle through all occurrences.

- 4.3 Replace Words in Source Code

This tool allows you to find all occurences of some search term and replace it with some alternative term. To use Search and Replace:

  1. Select Edit → Search and Replace. (Ctrl + H)
  2. A text box will appear prompting you to enter a search term and a replacement term.
  3. After entering both search and replacement terms, click Find to jump to its first occurrence within your source code.
  4. Click Replace to replace the highlighted search term.
  5. Continue clicking Find and Replace to cycle through and replace all instances of the search term.

- 4.4 Importing

This tool allows you to import text files.
To import a text file:

  1. Select File → Import.
  2. Choose which .txt or .rtf file you would like to import.

- 4.5 Exporting

This tool allows you to export source code to a text file or a rich text format file.

  1. Select File → Export.
  2. Choose a location for your exported file.

5.0 Options

ASSIST/UNA gives you the ability to change several options to better suit your needs. To access the options menu:

  1. Select Tools → Options.
  2. The Options Dialog Window appears displaying all ASSIST/I and ASSIST/UNA options.
